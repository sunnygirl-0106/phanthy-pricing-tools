⑥ 公式讲解（为什么是这样）

① 混合成本 c — 掺水到底省在哪
c = f·c_fast + (1−f)·c_std 视频有两种货：标准（贵，c_std）和 fast（便宜，c_fast）。按比例 f 掺 fast，整体进货成本就是两者的加权平均。掺得越多（f↑），c 越往 c_fast 靠、越便宜。 换个写法更直观：c = c_std − f·(c_std − c_fast)，即每多掺 1% 的水，成本就降 (c_std−c_fast)/100。这就是"掺水"省钱的全部来源——后台随机把一部分任务跑 fast，用户无感。
② 每秒毛利 & ③ 毛利率 — 同一个卖价怎么从亏变赚
每秒毛利 = p − c　　毛利率 = 1 − c/p 卖价 p 不变，但掺水把 c 压下去了，所以 p−c 从负变正。这就是关键：不是涨价，是降成本。例如纯标准成本 1.05、想卖 0.95 是亏的（0.95<1.05）；掺水把成本压到 0.95 以下，同样卖 0.95 就赚了。
④ 甜区 — "既比 libtv 便宜，又不亏"的唯一窗口
要同时满足两件事：
不亏 → p > c；比竞品便宜 → p < libtv。 合起来就是 c < p < libtv(0.86)。 这个窗口只有当 c < 0.86 时才存在。换句话说：掺水必须把成本压到 0.86 以下，否则"又便宜又赚"这件事数学上不成立——只能二选一。数轴上那条绿色虚线区就是甜区，卖价点（黄）落在里面才算赢。
⑤ 反解 f — 这条最实用
已知想卖 p、想保住毛利率 m，反推要掺多少水：
先由毛利率定成本上限：c = p·(1−m) 再由 c 反解 f（把①的第二种写法倒过来）：f = (c_std − c) / (c_std − c_fast) = (c_std − p·(1−m)) / (c_std − c_fast) 如果算出来 f > 100%：光靠掺水做不到，得去压底价（找补贴/谈进价）；如果 f < 0：说明不用掺水，纯标准就已经够赚。
⚠️ v2 更新 — fast 官方降到 0.63 后变好了多少
2.0 把 fast 成本打了 0.75 折(不含视频输入 720P:0.84 → 0.63),所以老板之前口算的那个 0.63 现在成了官方价。掺 20% 后混合成本 c=0.966(旧 1.008)。
效果:卖 0.95 的亏从每秒 5.8 分收窄到约 1.6 分;想卖 0.95 还保 5% 毛利,需要的掺水从 70% 降到约 35%——门槛大幅降低。想真转正,把掺水再提一点、或走含视频输入链路(c_fast=0.378)即可。这台计算器的意义就是:别把旋钮焊死在乐观位,把每种可能扫一遍。
